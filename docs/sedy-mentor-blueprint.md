# Šedý Mentor Homepage Funnel Spec

## Goal

The homepage is not a traditional homepage.
It is a direct funnel.

Primary goal:
move users into the system and toward the right zone

Secondary goal:
make the project feel like a structured product, not like random content

The homepage should be built for:
- social traffic
- warm curiosity traffic
- users coming from IG / TT / ManyChat / direct links

It should not be designed like:
- a blog
- an agency site
- a personal portfolio
- a typical course seller page

---

## Funnel Strategy

Homepage logic:
1. clear hook
2. clear problem
3. clear philosophical solution
4. zone progression
5. product proof
6. pricing / access logic
7. final CTA

Recommended CTA model:
hybrid

Primary CTA:
join Grey Zone / unlock the main system

Secondary CTA:
start free in White Zone

Reason:
social traffic is warm, but not always ready to buy immediately

---

## Core Messaging

Main brand line:
Zkušenost → Dovednost → Možnost

Main homepage message:
the system helps users turn digital knowledge into practical options

The homepage must communicate:
- this is a system
- this is practical
- this is not hype
- this is deeper than typical motivational content
- there is progression inside

---

## Section Structure

### 1. Hero Section

Purpose:
- instantly explain what Šedý Mentor is
- make the project feel like a system
- give a clear action path

Recommended headline direction:
Rozšiř si možnosti.
Nauč se rozumět systému dřív, než tě pohltí.

Recommended subheadline direction:
Šedý Mentor je uzavřený systém pro lidi, kteří nechtějí jen konzumovat obsah — chtějí získat zkušenost, dovednost a skutečnou možnost.

Primary CTA:
Vstoupit do Šedé zóny

Secondary CTA:
Začít zdarma v Bílé zóně

Hero should visually feel:
- premium
- minimal
- structured
- calm
- not overdesigned

---

### 2. Problem Section

Purpose:
show the current trap most users are in

Messaging direction:
- most people collect information
- most people consume content
- most people save hacks and tips
- but they do not build real options

Recommended angle:
Informace bez systému jsou jen zábava.

This section should make users feel:
- recognized
- slightly confronted
- open to a new model

---

### 3. Philosophy Section

Purpose:
introduce the brand’s core framework

Heading direction:
Zkušenost → Dovednost → Možnost

Content logic:
- experience opens the door
- skill gives the key
- possibility creates leverage

This section should explain that:
the goal is not perfect mastery
the goal is usable depth

---

### 4. Zone Progression Section

Purpose:
present the product architecture as a progression model

This section should visually show:
- Bílá Zóna
- Šedá Zóna
- Černá Zóna

It should feel like:
- level progression
- system depth
- access architecture

Not just:
- simple pricing cards

Each zone needs a short purpose:

#### Bílá Zóna
start, onboarding, current Drop, orientation

#### Šedá Zóna
tools, prompts, AI bots, Drop archive, main practical layer

#### Černá Zóna
founders access, internal updates, visual prompts, Project S7, deeper insider layer

CTA after this section:
Začít zdarma v Bílé zóně
or
Zobrazit Šedou zónu

---

### 5. What You Actually Get Section

Purpose:
show practical value in concrete terms

Recommended items:
- Šedá knihovna promptů
- Privátní AI boti
- Nástrojovna
- Video Downloader
- Šedý Archiv
- Vizuální prompty
- Dropy
- kurzy a mini-kurzy

Rules:
- concrete wording
- no empty marketing fluff
- no exaggerated claims

This section should make the system feel real and usable.

---

### 6. Who It Is For / Not For Section

Purpose:
increase trust and filter people

Recommended structure:

#### Je to pro tebe, pokud:
- nechceš být jen konzument obsahu
- hledáš efektivitu místo chaosu
- chceš rozumět systému a použít ho v praxi
- jsi ochotný testovat, myslet a budovat možnosti

#### Není to pro tebe, pokud:
- hledáš rychlé zbohatnutí
- chceš jen kopírovat bez pochopení
- nechceš přemýšlet nebo něco zkoušet
- hledáš další internetový hype projekt

This section should feel honest and filtering, not arrogant.

---

### 7. Pricing / Access Section

Purpose:
turn interest into decision

Recommended heading direction:
Vyber si úroveň, ve které chceš hrát.

Show:
- Bílá Zóna
- Šedá Zóna
- Černá Zóna

Use the current benefits structure.

Rules:
- White should feel like entry
- Grey should feel like main value
- Black should feel like founders / insider depth

Grey should visually feel like the main recommendation.
Black should feel premium, not mainstream.

---

### 8. Final CTA Section

Purpose:
give a final decisive next step

Recommended message direction:
Nejde o motivaci.
Jde o systém.

Recommended CTA options:
- Vstoupit do Šedé zóny
- Začít zdarma v Bílé zóně

Keep this section short and clean.

---

## Mobile Logic

The homepage must be mobile-first.

Rules:
- visual first
- short sections
- clear CTA repetition
- avoid long text walls
- spacing must feel premium
- cards and sections must be easy to scan

Important:
hero should not overload mobile users with too much text at once

---

## Navigation Rules

Recommended:
- minimal or no top menu
- keep users inside funnel logic
- reduce distractions
- repeat CTA through the page

Avoid:
- blog-like navigation
- too many paths
- unnecessary links out of the funnel

---

## Design Rules

The homepage should visually reflect:
White → Grey → Black progression

Recommended design feeling:
- premium minimalism
- product-first layout
- clear cards
- restrained accent usage
- dark/light contrast as a progression device

Do not design it like:
- an info portal
- a typical WordPress blog
- a noisy marketing page

---

## Conversion Principles

The homepage must support:
- trust
- clarity
- curiosity
- progression
- low friction entry
- upgrade logic

Do not use:
- fake scarcity
- loud countdown style tactics
- empty bonus stacking
- exaggerated guru promises

Use:
- system logic
- clean structure
- concrete value
- smart progression
- calm confidence

---

## Key Psychological Effect

Users should feel:
- this is organized
- this is useful
- this is deeper than typical internet content
- this is a system worth entering
- there is more inside than what is shown outside

They should not feel:
- pushed
- manipulated
- overwhelmed
- sold to by a fake expert

---

## Build Direction

If building a prototype:
- build homepage as a direct funnel
- use section-based structure
- use clear CTA hierarchy
- make Grey Zone the main value anchor
- preserve White as a free entry path
- preserve Black as founders / insider depth

The final result should feel like:
a premium digital system with clear internal architecture and long-term brand discipline
